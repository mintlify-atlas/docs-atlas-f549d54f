ValveResourceFormat supports over 50 different Source 2 resource types, each serving a specific purpose in game development. Resources are identified by their file extension and contain specialized data structures.

## Core resource categories

Source 2 resources are organized into several major categories based on their function.

### Models and animation

Resources related to 3D models, skeletal animation, and character behavior.

| Extension | Resource type | Description |
|-----------|---------------|-------------|
| `vmdl` | Model | 3D model created in the ModelDoc Editor |
| `vmesh` | Mesh | Mesh geometry (legacy, stored in vmdl) |
| `vmorf` | Morph | Morph sets for blend shapes and facial animation |
| `vphys` | PhysicsCollisionMesh | Physics collision mesh data |

### Animgraph 1 (legacy)

| Extension | Resource type | Description |
|-----------|---------------|-------------|
| `vanmgrph` | AnimationGraph | Graph for animation behavior and states |
| `vanim` | Animation | Animation file (legacy, stored in vmdl) |
| `vagrp` | AnimationGroup | Animation group (legacy, stored in vmdl) |
| `vseq` | Sequence | Sequence group (legacy, stored in vmdl) |

### Animgraph 2

The newer animation system with enhanced capabilities.

| Extension | Resource type | Description |
|-----------|---------------|-------------|
| `vnmgrph` | NmGraph | Animation behavior graph |
| `vnmvar` | NmGraphVariation | Graph variations with different parameters |
| `vnmskel` | NmSkeleton | Skeleton and bonemask data |
| `vnmclip` | NmClip | Animation clip with event support |
| `vnmikrig` | NmIKRig | Inverse kinematics rig data |

### Textures and materials

| Extension | Resource type | Description |
|-----------|---------------|-------------|
| `vtex` | Texture | Compiled texture in various formats |
| `vmat` | Material | Material definition from Material Editor |
| `vcompmat` | CompositeMaterial | Composite material kit (CS2 weapon skins) |

### Particles and effects

### Maps and worlds

Level geometry, entities, and world organization.

| Extension | Resource type | Description |

### Sound

## Determining resource type

ValveResourceFormat determines resource types through multiple methods:

1. **File extension** - The primary method, mapped through the `ResourceType` enum
2. **Compiler identifiers** - Found in the REDI/RED2 block's special dependencies
3. **Input dependencies** - When there's a single input dependency, its extension is used

```csharp
// Example from Resource.cs:203
ResourceType = ResourceTypeExtensions.DetermineByFileExtension(
    Path.GetExtension(FileName.AsSpan())
);
```

<Note>
Some resource types like `vmesh`, `vanim`, and `vagrp` are marked as legacy because their data is now stored directly in the `vmdl` (model) file.
</Note>

## Support status

Most resource types are fully supported (👍), with a few exceptions:

- `vrmap` (Resource Remap Table) - Not supported
- `vseq` (Sequence Group) - Not supported
- `vsurf` (Surface Properties) - Not supported
- `vvis` (World Visibility) - Not supported

Unsupported resources can still be parsed for basic information, but specialized data extraction is not available.

## Resource type enum

Resource types are defined in `ResourceType.cs` with friendly names and extension attributes:

```csharp
[Extension("vmdl")]
Model,

[Extension("vtex")]
Texture,

[Extension("vmat")]
Material,
```
